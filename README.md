# Coding Test Practice

This repository is organized for LeetCode `Easy -> Medium` preparation with two main study areas:

- `notes/`: concept notes and reusable Python templates
- `problems/`: problem files for structured practice

## Problem Areas

### `problems/basic`

Core LeetCode practice grouped by topic and difficulty. Each problem file includes a header template with:

- problem metadata
- weekly study plan
- idea / complexity notes
- review fields for `Status`, `Mistake`, and `Retry`

### `problems/practical`

Future folders for broader coding-test practice outside the core basic track:

- `programmers/`
- `baekjoon/`
- `leetcode-mixed/`

### `problems/advanced`

Future folders for deeper topic-specific training:

- `graph/`
- `dp/`
- `string/`
- `data-structure/`

## Weekly Routine

- 4 days per week
- Day 1: concept study + one light example
- Day 2~4: problem solving
- Weekly target: 10 problems

Recommended flow:

1. Read the matching note in `notes/`.
2. Solve the Day 1 concept example to check the pattern.
3. Work through Day 2~4 target problems.
4. Update `Status`, `Mistake`, and `Retry` inside each problem file header right after solving.

## Repository Structure

```text
coding-test-practice/
├─ README.md
├─ roadmap.md
├─ notes/
└─ problems/
   ├─ basic/
   ├─ practical/
   └─ advanced/
```

## Review Method

There is no separate review directory in this structure.

Keep review notes directly inside each problem file header:

- `Status`: `Unsolved`, `Solved`, `Hint`, or `Failed`
- `Mistake`: what went wrong
- `Retry`: when to revisit the problem

## Study Rule

- Do not focus only on accepted answers.
- Write the pattern choice first, then the implementation.
- Retry weak problems after a short gap instead of rereading the full solution immediately.
